# 📋 Alcance y Limitaciones de la Licencia MIT

## 📖 Overview

La **Licencia MIT** es una de las licencias de código abierto más permisivas y simples. Permite a casi cualquiera hacer casi cualquier cosa con el código de Threshold, con muy pocas restricciones.

---

## ✅ Qué SÍ Puedes Hacer

### 1. **Usar Comercialmente**
- Puedes usar Threshold en proyectos comerciales
- Puedes crear un startup basado en Threshold
- Puedes vender software que use Threshold como base
- **Ejemplo:** Crear una plataforma de educación online usando el código

### 2. **Modificar el Código**
- Puedes cambiar, eliminar o agregar funcionalidades
- Puedes refactorizar y optimizar
- Puedes adaptar a tus necesidades específicas
- **Ejemplo:** Cambiar el algoritmo [[spaced_repetition_logic|SM-2]] por otro personalizado

### 3. **Distribuir (Copiar y Compartir)**
- Puedes copiar el código y compartirlo
- Puedes distribuirlo en redes, blogs, repositorios
- Puedes incluirlo en otros proyectos
- **Ejemplo:** Clonar el repo y usarlo como base para tu tesis

### 4. **Crear Versiones Privadas**
- Puedes hacer cambios sin compartirlos públicamente
- Puedes tener ramas privadas
- Puedes modificar sin reportar cambios
- **Ejemplo:** Fork privado con mejoras propias

### 5. **Sublicenciar**
- Puedes usar Threshold como parte de otro software
- Puedes redistribuir bajo otra licencia (compatible)
- Puedes integrar en productos cerrados
- **Ejemplo:** Usar components de Threshold en una app de pago

### 6. **Usar sin Permiso Previo**
- No necesitas pedir autorización a Oponobono
- No necesitas hacer fork oficial
- Puedes empezar a usar inmediatamente
- **Ejemplo:** Descargar y comenzar a trabajar sin contactar al autor

---

## ❌ Qué NO Puedes Hacer

### 1. **Reclamar Autoría**
❌ **NO PUEDES:** Pretender que escribiste el código original
- No puedes borrar la mención de Oponobono
- No puedes cambiar los créditos
- No puedes presentar como tuyo en conferencias/artículos sin aclarar

✅ **SÍ DEBES:** Mantener el aviso de copyright original

### 2. **Usar sin la Licencia**
❌ **NO PUEDES:** Distribuir Threshold sin incluir la licencia MIT
- Si redistribuyes el código, DEBES incluir el archivo LICENSE
- DEBES incluir el aviso de copyright
- DEBES explicar cambios si modificas

✅ **SÍ DEBES:** Copiar el texto de MIT License cuando distribuyas

### 3. **Responsabilidad por Daños**
❌ **NO PUEDES:** Demandar a Oponobono si algo falla
- El software se proporciona "AS IS" (como está)
- Sin garantía de ningún tipo
- Sin responsabilidad por pérdidas

✅ **ENTIENDES QUE:** El código viene sin garantía

### 4. **Mantener Secretos del Licenciante**
❌ **NO PUEDES:** Usarlo para acceder/copiar secretos de Oponobono
- No puedes usar como herramienta de espionaje
- No puedes violar privacidad
- No puedes acceder datos no autorizados

✅ **DEBES:** Respetar privacidad y leyes

## 🔑 Condiciones Clave

### 1. **Incluir Licencia**
Siempre que distribuyas Threshold (modificado o no), **DEBES:**
- ✅ Incluir copia del archivo LICENSE
- ✅ Incluir aviso de copyright (Copyright © 2026 Oponobono)
- ✅ Incluir la licencia MIT completa

**Ubicación típica:**
```
TU_PROYECTO/
├── LICENSE                 ← Copia de la licencia MIT
├── README.md              ← Menciona "Basado en Threshold"
└── src/...
```

### 2. **Avisos de Cambios**
Si modificas, **es recomendable (no obligatorio):**
- 📝 Documentar qué cambiaste
- 📝 Incluir archivo CHANGELOG
- 📝 Agregar comentarios en código modificado

**Ejemplo:**
```javascript
// Código original de Threshold
// Modificado en 2026 para XYZ
```

### 3. **Mantener Aviso de Copyright**
En archivos principales, **SÍ DEBES:**
```
/*
 * Threshold - Basado en código de Oponobono (MIT License)
 * Copyright © 2026 Oponobono
 * Modifications © 2026 Tu Nombre
 */
```

   - Crear fork mejorado

   - Crear app similar basada en Threshold
   - Mejorar y ofrecer versión propia
   - Cambiar nombre y branding

## ❌ Casos NO Permitidos

### ❌ Prohibido

1. **Plagio Intelectual**
   - Publicar como propio sin mencionar Oponobono
   - En conferencia: "Yo creé este algoritmo"
   - En artículo: No citar la fuente

2. **Distribución sin Licencia**
   - Vender en App Store sin incluir LICENSE
   - Hacer fork cerrado y no incluir MIT
   - Copiar en tu README sin mencionar

3. **Garantías Falsas**
   - Prometer que es 100% confiable
   - Garantizar funcionamiento perfecto
   - Vender como "producto garantizado"

4. **Violación de Privacidad**
   - Usar para minería de datos de usuarios
   - Acceder información confidencial
   - Usar para phishing o fraude
